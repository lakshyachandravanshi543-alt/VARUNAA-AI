🔬 How AquaSense AI Works

AquaSense AI bridges the gap between hardware engineering and artificial intelligence. Instead of relying on extremely expensive physical chemical analyzers (which often cost upwards of $10,000 and degrade quickly in harsh river conditions), this platform uses only 4 basic, low-cost physical probes:

  • pH: Measures Acidity & Alkalinity
  • Dissolved Oxygen (DO): The lifeline of aquatic biology
  • Turbidity: Measures water cloudiness and suspended solids
  • Temperature: Governs chemical reaction speeds

By feeding these simple hardware readings into a Machine Learning model, the AI reconstructs the "chemical fingerprint" of the river to deduce complex, hidden impurities (like heavy metals, raw sewage, and oil slicks) that usually require a laboratory to detect.

⚙️ The Physical Hardware Node

To successfully capture real-time river data, the system relies on a low-power, waterproof hardware node designed for prolonged remote deployment. This is the physical "brain" of the operation that lives directly in the river.

  • Microcontroller: ESP32 Processor (Chosen for ultra-low power consumption and built-in Wi-Fi/Bluetooth stacks).
  • Analog Interface: Probes are connected via raw analog pins (e.g., pH on Pin 34, Turbidity on Pin 35) taking raw voltage readings from the water stream.
  • Power Supply: Designed to run continuously on LiFePO4 batteries coupled with a 5W Solar Panel for off-the-grid survival.

📡 Hardware-to-Cloud Telemetry

Reading raw sensor voltage is only step one. The system must transport this data instantly securely out of the river and into the Machine Learning engine located in Google Cloud.

  1. Polling: The C++ code on the ESP32 wakes up every 10 seconds and averages 10 rapid readings to filter out electrical noise.
  2. Calibration: The raw voltages (0-4095) are mathematically scaled on the device (e.g., converting voltage curve to NTU for turbidity).
  3. JSON Payload: The Arduino C++ code binds the 4 finalized metrics into a secure JSON Payload wrapper.
  4. POST Request: The node shoots the payload via HTTP POST directly to our deployed Google Cloud Run API endpoint where the AI scaler intercepts it!
